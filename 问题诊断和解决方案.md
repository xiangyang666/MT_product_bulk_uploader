# 问题诊断和解决方案

## 当前问题

exe 文件上传到 100% 后出现 `ERR_CONNECTION_RESET` 错误。

### 错误日志分析
```
美团商品上传工具-1.0.0-x64.exe 上传进度: 100%
响应错误: AxiosError: Network Error
POST http://106.55.102.48:8080/api/app-versions/upload net::ERR_CONNECTION_RESET
```

### 问题原因

1. **文件已完整上传** ✅ - 进度显示 100%，说明文件已经传输完成
2. **服务器处理时断开** ❌ - 服务器在保存文件到 MinIO 时连接被重置

可能的原因：
- Nginx 配置未更新（最可能）
- 后端处理超时
- MinIO 服务未运行或配置错误
- 服务器资源不足（内存/磁盘）

## 解决方案

### 方案 1：更新 Nginx 配置（必须执行！）

**这是最重要的步骤，必须先完成！**

#### 选项 A：自动脚本（推荐）
```bash
# 在本地电脑执行
ssh root@106.55.102.48 'bash -s' < 快速修复-Nginx配置.sh
```

#### 选项 B：手动操作
```bash
# 1. SSH 登录
ssh root@106.55.102.48

# 2. 备份配置
cp /etc/nginx/sites-available/default /etc/nginx/sites-available/default.backup

# 3. 编辑配置
vi /etc/nginx/sites-available/default

# 4. 删除所有内容，复制 nginx-config-optimized.conf 的内容进去

# 5. 保存退出（ESC 然后 :wq）

# 6. 测试配置
nginx -t

# 7. 重启 Nginx
systemctl restart nginx
```

## 技术细节

### Nginx 配置关键点

1. **client_max_body_size 500M**
   - 允许上传 500MB 文件
   - 默认只有 1MB

2. **proxy_read_timeout 600s**
   - 代理读取超时 10 分钟
   - 给服务器足够时间处理大文件

3. **proxy_request_buffering off**
   - 禁用代理缓冲
   - 对大文件上传至关重要
   - 避免 Nginx 缓冲整个文件

4. **反向代理配置**
   - 将 /api/ 请求转发到 localhost:8080
   - 之前的配置缺少这个，导致请求无法到达后端

### 为什么上传到 100% 后才失败？

1. 前端上传进度 100% = 文件已传输到 Nginx
2. Nginx 需要将文件转发给后端（8080端口）
3. 后端需要保存文件到 MinIO
4. 如果 Nginx 配置不正确，在这个阶段会断开连接

### 流程图
```
客户端 -> Nginx -> 后端 -> MinIO
  ✓       ✗       ?       ?
(100%)  (断开)
```

修复 Nginx 配置后：
```
客户端 -> Nginx -> 后端 -> MinIO
  ✓       ✓       ✓       ✓
(100%)  (转发)  (处理)  (保存)
```
