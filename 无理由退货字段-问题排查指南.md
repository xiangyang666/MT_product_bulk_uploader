# 无理由退货字段问题排查指南

## 问题描述

**现象**：导入"7天无理由退货（一次性包装破损不支持）"，导出却是 1300030895（不支持7天无理由退货）

**期望**：应该导出 1300030902

## 根本原因

之前的代码有一个有问题的兜底逻辑：

```java
// 如果包含"不支持"但不是上述具体情况，默认为不支持
if (trimmedValue.contains("不支持")) {
    return "1300030895";
}
```

这导致 "7天无理由退货（一次性包装破损**不支持**）" 在精确匹配失败后，被错误地匹配到 1300030895。

## 修复方案

### 1. 移除有问题的兜底逻辑

已删除 `if (trimmedValue.contains("不支持"))` 这个逻辑。

### 2. 添加模糊匹配作为备选

为每个标签添加关键词匹配，以防精确匹配因字符编码问题失败：

```java
// 1300030902 - 7天无理由退货（一次性包装破损不支持）
if (trimmedValue.equals("7天无理由退货（一次性包装破损不支持）") ||
    trimmedValue.equals("7天无理由退货(一次性包装破损不支持)") ||
    trimmedValue.contains("一次性包装破损")) {  // 添加关键词匹配
    return "1300030902";
}
```

### 3. 添加调试日志

添加了 `log.debug()` 和 `log.warn()` 来帮助排查问题：

```java
log.debug("匹配到: 7天无理由退货（一次性包装破损不支持）-> 1300030902");
log.warn("无理由退货字段未匹配到任何标签，使用默认值。输入值: [{}], 长度: {}", trimmedValue, trimmedValue.length());
```

## 常见问题

### Q1: 为什么精确匹配会失败？

A: 可能的原因：
- Excel中的字符编码不一致
- 复制粘贴时引入了不可见字符
- 全角/半角字符混用
- 空格字符不一致

### Q2: 关键词匹配会不会误匹配？

A: 不会。关键词是唯一的：
- "一次性包装破损" 只出现在 1300030902
- "激活后" 只出现在 1300030903
- "使用后" 只出现在 1300030904
- "安装后" 只出现在 1300030905
- "定制类" 只出现在 1300030906

### Q3: 如果还是匹配错误怎么办？

A: 查看后端日志中的警告信息：
```
无理由退货字段未匹配到任何标签，使用默认值。输入值: [XXX], 长度: XX
```

将输入值和长度信息提供给技术支持。
