## 🎯 问题解决

### 原始问题
美团更新了新模板v1.3.7，列名发生变化：
- v1.3.6: "重量" → weight
- v1.3.7: "毛重" → weight

现有系统使用固定映射，无法识别新版本的列名。

### 解决方案
实现了自适应字段映射系统，通过4种匹配算法自动识别列名变体：
1. **精确匹配** - 列名完全相同（置信度1.0）
2. **标准化匹配** - 去除空格、标点后匹配（置信度0.95）
3. **关键词匹配** - 包含关键词组合（置信度0.8-0.9）
4. **模糊匹配** - 字符串相似度（置信度0.5-0.8）

## 📦 已完成的核心组件

### 1. 数据模型（任务1）
- `MatchType` - 匹配类型枚举
- `FieldMappingRule` - 字段映射规则
- `ColumnMapping` - 列映射结果
- `MappingWarning` - 映射警告
- `MappingError` - 映射错误
- `MappingResult` - 完整映射结果

### 2. 语义匹配器（任务2）
- `SemanticMatcher` - 实现4种匹配算法
- 支持精确、标准化、关键词、模糊匹配
- 使用Levenshtein距离算法计算字符串相似度

### 3. 置信度计算器（任务3）
- `ConfidenceCalculator` - 计算映射置信度
- 基于匹配类型返回0-1的置信度
- 支持优先级调整和历史数据调整（预留）

### 4. 映射规则配置（任务4）
- `mapping-rules.json` - 30+个字段的映射规则
- 支持精确匹配、关键词、关键词组合、同义词
- **关键**：weight字段支持"重量"、"毛重"、"净重"

### 5. 映射规则仓库（任务5）
- `MappingRuleRepository` - 加载和管理映射规则
- 从JSON配置文件加载规则
- 配置错误时使用内置默认规则
- 支持热加载（reloadRules方法）

### 6. 自适应映射引擎（任务6）
- `AdaptiveMappingEngine` - 核心协调器
- 构建字段映射、解决冲突、验证结果
- 生成警告（置信度<0.8）和错误（必需字段缺失）

### 7. 系统集成（任务7）
- 集成到`FormatDetector`
- 配置开关控制（默认启用）
- 智能回退到固定映射

## 📋 配置说明

### application.yml

```yaml
meituan:
  adaptive-mapping:
    enabled: true  # 启用自适应映射（默认true）
    config-file: classpath:mapping-rules.json
    confidence-threshold: 0.8  # 置信度阈值
```

### mapping-rules.json

```json
{
  "version": "1.0",
  "rules": {
    "weight": {
      "fieldName": "weight",
      "displayName": "重量",
      "required": false,
      "exactMatches": ["重量", "毛重", "净重"],  // 支持多个变体
      "keywords": ["重量", "毛重", "净重", "weight"],
      "synonyms": ["重", "克重", "分量"],
      "priority": 70
    },
    // ... 其他30+个字段
  }
}
```

## 🎨 核心特性

### 1. 自动识别列名变体
- "重量" / "毛重" / "净重" → weight
- "商品类目ID" / "商品三级类目ID" → categoryId
- "价格" / "价格（元）" → price

### 2. 智能匹配算法
```
精确匹配 (1.0) → 标准化匹配 (0.95) → 关键词匹配 (0.8-0.9) → 模糊匹配 (0.5-0.8)
```

### 3. 配置化管理
- 通过JSON文件管理映射规则
- 无需修改代码即可添加新的列名变体
- 支持热加载（无需重启）

### 4. 向后兼容
- 保持原有API接口不变
- 自适应映射失败时自动回退到固定映射
- 可通过配置禁用自适应映射

### 5. 详细日志

## 📊 技术架构

```
Excel表头
    ↓
FormatDetector.getMeituanColumnMapping()
    ↓
AdaptiveMappingEngine.buildMapping()
    ↓
    ├─ MappingRuleRepository.getAllRules()  // 加载规则
    ├─ SemanticMatcher.findBestMatch()      // 匹配算法
    ├─ ConfidenceCalculator.calculate()     // 计算置信度
    └─ resolveMappingConflicts()            // 解决冲突
    ↓
MappingResult
    - mappings: Map<String, ColumnMapping>
    - warnings: List<MappingWarning>
    - errors: List<MappingError>
    - valid: boolean
```

## 🔧 故障排查

### 问题1：映射失败
**症状**：日志显示"自适应映射失败，回退到固定映射"

**解决方案**：
1. 检查 `mapping-rules.json` 文件是否存在
2. 验证JSON格式是否正确
3. 查看详细错误日志

### 问题2：列名无法识别
**症状**：某些列无法映射

**解决方案**：
1. 在 `mapping-rules.json` 中添加新的列名变体
2. 重启服务或调用热加载接口

### 问题3：禁用自适应映射
**场景**：需要回退到固定映射

**操作**：
```yaml
meituan:
  adaptive-mapping:
    enabled: false  # 禁用自适应映射
```
