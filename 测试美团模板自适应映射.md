# 美团模板自适应映射 - 测试指南

## 测试目标

验证系统能够自动识别和处理不同版本的美团模板（v1.3.6和v1.3.7），特别是列名变化的情况。

## 测试环境

- 后端服务：`start-backend.bat`
- 前端服务：`start-frontend.bat`
- 测试文件位置：
  - v1.3.6: `资源/批量创建模板_v1.3.6.xlsx`
  - v1.3.7: `美团模板/批量创建模板_v1.3.7(1).xlsx`

## 映射置信度验证

### 高置信度映射（≥0.95）
- 精确匹配：列名完全相同
- 标准化匹配：去除空格、标点后相同

**示例：**
- "商品名称" → productName (1.0)
- "价格（元）" → price (0.95，标准化后为"价格元")

### 中等置信度映射（0.8-0.95）
- 关键词匹配：包含所有关键词

**示例：**
- "商品三级类目ID" → categoryId (0.85-0.9)

### 低置信度映射（0.5-0.8）
- 模糊匹配：字符串相似度

**预期：**
- 低置信度映射应该生成警告
- 但仍然可以使用

## 配置验证

### 检查配置文件

**application.yml:**
```yaml
meituan:
  adaptive-mapping:
    enabled: true  # 确保启用
    config-file: classpath:mapping-rules.json
    confidence-threshold: 0.8
```

**mapping-rules.json:**
```json
{
  "rules": {
    "weight": {
      "exactMatches": ["重量", "毛重", "净重"],
      ...
    }
  }
}
```

## 回退测试

### 禁用自适应映射

1. 修改 `application.yml`:
```yaml
meituan:
  adaptive-mapping:
    enabled: false
```

2. 重启后端服务

3. 上传v1.3.7模板

**预期结果：**
- ✅ 系统使用固定映射
- ✅ "毛重"列无法映射（因为固定映射只支持"重量"）
- ⚠️ 可能出现字段缺失警告

### 配置文件错误处理

1. 重命名 `mapping-rules.json` 为 `mapping-rules.json.bak`

2. 重启后端服务

**预期结果：**
- ⚠️ 日志显示：配置文件不存在，使用默认规则
- ✅ 系统使用内置的6个核心字段默认规则
- ✅ 基本功能仍然可用

## 故障排查

### 问题1：映射失败

**症状：**
- 日志显示"自适应映射失败，回退到固定映射"

**检查：**
1. 确认 `mapping-rules.json` 文件存在
2. 检查JSON格式是否正确
3. 查看详细错误日志

### 问题2：列名无法识别

**症状：**
- 某些列无法映射

**解决方案：**
1. 检查 `mapping-rules.json` 中是否包含该列名的规则
2. 添加新的列名变体到配置文件
3. 重启服务或调用热加载接口

### 问题3：置信度过低

**症状：**
- 映射成功但置信度 < 0.8，产生警告

**解决方案：**
1. 在 `mapping-rules.json` 中添加精确匹配规则
2. 或者调整 `confidence-threshold` 配置
