# 清除商品 502 错误 - 解决方案

## 🐛 问题描述

点击"清除美团所有商品"按钮时出现 502 Bad Gateway 错误：
```
DELETE http://localhost:5173/api/products/clear 502 (Bad Gateway)
```

## 🔍 问题原因

后端服务器没有运行，导致前端无法连接到后端 API。

## ✅ 解决方案

### 启动后端服务

使用以下任一方法启动后端：

#### 方法 1：使用批处理文件（推荐）
```bash
start-backend.bat
```

#### 方法 2：手动启动
```bash
cd meituan-backend
mvn spring-boot:run
```

#### 方法 3：启动所有服务
```bash
start-all.bat
```

### 验证服务启动

1. 等待后端启动完成（通常需要 10-30 秒）
2. 查看控制台输出，确认看到类似信息：
   ```
   Started MeituanProductApplication in X.XXX seconds
   ```
3. 访问后端健康检查（如果有）：
   ```
   http://localhost:8080/actuator/health
   ```

### 重新测试清除功能

1. 确认后端服务已启动
2. 刷新前端页面（Ctrl + Shift + R）
3. 点击"清除美团所有商品"按钮
4. 输入访问令牌（默认：admin123）
5. 确认清除操作

## 📋 后端接口信息

- **端点**：`DELETE /api/products/clear`
- **请求体**：
  ```json
  {
    "merchantId": 1,
    "accessToken": "admin123"
  }
  ```
- **响应**：
  ```json
  {
    "code": 200,
    "message": "成功清空X条商品",
    "data": {
      "deletedCount": X
    }
  }
  ```

## 🔧 常见问题

### 1. 后端启动失败
- 检查 Java 版本（需要 Java 8 或更高）
- 检查 Maven 是否正确安装
- 检查数据库连接配置

### 2. 端口被占用
- 检查 8080 端口是否被其他程序占用
- 修改 `application.yml` 中的端口配置

### 3. 数据库连接失败
- 确认 MySQL 服务正在运行
- 检查数据库连接配置（用户名、密码、数据库名）

---

**问题类型**：后端服务未启动  
**解决方案**：启动后端服务  
**状态**：需要启动后端
