# 需求文档 - 美团模板自适应字段映射

## 简介

美团会不断更新其导出模板（如从v1.3.6到v1.3.7），列名可能会发生变化。系统应该能够自适应识别不同版本的美团模板，自动匹配字段，而不是依赖固定的列名映射。

## 术语表

- **美团模板版本**：美团后台导出模板的版本号，如v1.3.6、v1.3.7
- **自适应映射**：系统根据列名的语义自动匹配到系统字段，而不是依赖精确的列名
- **模糊匹配**：通过关键词、同义词、相似度等方式匹配列名
- **映射规则库**：存储不同字段的可能列名变体的规则集合
- **映射置信度**：系统对某个列名映射的确信程度（0-1之间）

## 需求

### 需求 1：智能列名识别

**用户故事**：作为商家，我想系统能够自动识别不同版本美团模板的列名变化，这样我就不需要等待系统更新就能导入新版本模板的数据。

#### 验收标准

1. WHEN 系统读取Excel表头 THEN 系统应对每个列名进行语义分析
2. WHEN 列名包含"商品名称"、"名称"、"商品名"等关键词 THEN 系统应识别为商品名称字段
3. WHEN 列名包含"类目"、"分类"、"category"等关键词且包含"ID"或"编号" THEN 系统应识别为类目ID字段
4. WHEN 列名包含"价格"、"单价"、"price"等关键词 THEN 系统应识别为价格字段
5. WHEN 列名包含"库存"、"数量"、"stock"等关键词 THEN 系统应识别为库存字段
6. WHEN 列名包含"描述"、"详情"、"说明"等关键词 THEN 系统应识别为商品描述字段

### 需求 2：多版本模板支持

**用户故事**：作为商家，我想系统能够同时支持美团的多个模板版本，这样我就能导入不同时期导出的数据。

#### 验收标准

1. WHEN 系统检测到v1.3.6模板（包含"商品类目ID"列） THEN 系统应正确映射所有字段
2. WHEN 系统检测到v1.3.7模板（列名可能有变化） THEN 系统应正确映射所有字段
3. WHEN 系统检测到未知版本的美团模板 THEN 系统应尝试通过语义匹配识别字段
4. WHEN 多个列名都可能匹配同一字段 THEN 系统应选择置信度最高的列
5. WHEN 系统无法确定某个必需字段的映射 THEN 系统应提示用户手动指定

### 需求 3：映射规则配置化

**用户故事**：作为系统管理员，我想能够配置字段映射规则，这样当美团再次更新模板时，我可以快速添加新的列名变体而不需要修改代码。

#### 验收标准

1. WHEN 系统启动 THEN 系统应从配置文件加载字段映射规则
2. WHEN 配置文件包含新的列名变体 THEN 系统应能够识别这些新变体
3. WHEN 配置文件格式错误 THEN 系统应使用默认的内置规则并记录警告
4. WHEN 管理员更新配置文件 THEN 系统应能够热加载新规则（或重启后生效）
5. WHEN 配置文件不存在 THEN 系统应使用默认的内置规则

### 需求 4：映射结果可视化

**用户故事**：作为商家，我想在导入前看到系统是如何映射字段的，这样我就能确认映射是否正确。

#### 验收标准

1. WHEN 用户上传文件进行预览 THEN 系统应显示字段映射结果
2. WHEN 显示映射结果 THEN 系统应显示"美团列名 → 系统字段"的对应关系
3. WHEN 某个字段映射置信度较低（<0.8） THEN 系统应用警告标识显示该映射
4. WHEN 某个必需字段未能映射 THEN 系统应用错误标识显示并提示用户
5. WHEN 用户对映射不满意 THEN 系统应提供手动调整映射的选项

### 需求 5：映射学习和优化

**用户故事**：作为系统，我想记录成功的映射案例，这样我就能不断优化映射准确率。

#### 验收标准

1. WHEN 用户成功导入数据 THEN 系统应记录本次使用的列名和映射关系
2. WHEN 系统记录映射案例 THEN 系统应包含列名、映射字段、导入时间、成功行数
3. WHEN 某个列名出现频率较高 THEN 系统应提高该列名的映射优先级
4. WHEN 用户手动调整映射 THEN 系统应记录用户的选择作为学习样本
5. WHEN 系统积累足够的映射案例 THEN 系统应能够自动建议添加新的映射规则

### 需求 6：向后兼容性

**用户故事**：作为开发者，我想确保新的自适应映射功能不影响现有的固定映射功能，这样已有的导入流程不会被破坏。

#### 验收标准

1. WHEN 系统检测到精确匹配的列名 THEN 系统应优先使用精确匹配而不是模糊匹配
2. WHEN 文件使用标准格式 THEN 系统应按照原有逻辑处理
3. WHEN 文件使用旧版本美团格式 THEN 系统应能够正确识别和处理
4. WHEN 自适应映射失败 THEN 系统应回退到固定映射逻辑
5. WHEN 系统更新映射规则 THEN 已有的导入功能应继续正常工作

### 需求 7：映射性能优化

**用户故事**：作为商家，我想快速完成字段映射，即使文件有很多列也不应该影响导入速度。

#### 验收标准

1. WHEN 文件包含100列 THEN 字段映射应在1秒内完成
2. WHEN 系统进行模糊匹配 THEN 系统应使用高效的字符串匹配算法
3. WHEN 系统加载映射规则 THEN 系统应缓存规则避免重复加载
4. WHEN 系统计算映射置信度 THEN 系统应使用轻量级的评分算法
5. WHEN 文件很大（3000+行） THEN 字段映射不应显著增加总处理时间

## 非功能性需求

### 性能要求
- 字段映射时间 ≤ 1秒（100列以内）
- 映射规则加载时间 ≤ 100毫秒
- 不应显著增加现有导入流程的处理时间

### 准确性要求
- 精确匹配准确率 = 100%
- 模糊匹配准确率 ≥ 95%（对于常见的列名变体）
- 必需字段识别率 ≥ 98%

### 可用性要求
- 映射结果清晰易懂
- 警告和错误信息明确
- 支持手动调整映射

### 可维护性要求
- 映射规则配置化，易于更新
- 映射逻辑独立，易于测试
- 完整的映射日志，便于问题排查

### 可扩展性要求
- 易于添加新的字段类型
- 易于添加新的匹配算法
- 易于支持其他平台的模板格式
