### 后端提交 (commit: 22ef5b4)
**标题**：fix: 修复类目属性和无理由退货字段问题

**修改文件**：
- `meituan-backend/src/main/java/com/meituan/product/service/FormatDetector.java`
- `meituan-backend/src/main/java/com/meituan/product/service/MeituanFormatParser.java`
- `meituan-backend/src/main/resources/mapper/ProductMapper.xml`

**主要变更**：

#### 1. 类目属性修复
- ✅ 统一"类目属性"列映射到`productAttributes`字段
- ✅ 优化读取逻辑，优先使用列映射而非硬编码索引
- ✅ 添加诊断日志，警告类目属性为空或过短的情况

**修改详情**：
```java
// FormatDetector.java
MEITUAN_COLUMN_MAPPING.put("类目属性", "productAttributes");  // 之前映射到categoryAttributes

// MeituanFormatParser.java
// 优先使用列映射，硬编码索引作为备选
String productAttrs = extractField(row, "productAttributes", columnMapping);
if (productAttrs == null || productAttrs.trim().isEmpty()) {
    productAttrs = extractFieldByIndex(row, 50);
}

// 添加诊断日志
if (productAttrs == null || productAttrs.trim().isEmpty()) {
    log.warn("第{}行：类目属性为空！", rowNum);
} else if (productAttrs.length() < 50) {
    log.warn("第{}行：类目属性过短（{}字符），可能不完整", rowNum, productAttrs.length());
}
```

#### 2. 无理由退货字段修复
- ✅ 在`ProductMapper.xml`的INSERT语句中添加`no_reason_return_tag_id`字段
- ✅ 确保导入时解析的标签ID能正确存储到数据库
- ✅ 修复导出时标签ID为NULL导致使用默认值的问题

**修改详情**：
```xml
<!-- ProductMapper.xml -->
<insert id="batchInsert">
    INSERT INTO t_product (
        ..., no_reason_return, no_reason_return_tag_id, is_combo, ...
    ) VALUES
    <foreach collection="list" item="item" separator=",">
        (
            ..., #{item.noReasonReturn}, #{item.noReasonReturnTagId}, #{item.isCombo}, ...
        )
    </foreach>
</insert>
```

**影响范围**：
- 导入商品时，类目属性能正确读取和存储
- 无理由退货标签ID能正确存储到数据库
- 导出时能正确显示标签ID（1300030895-1300030906, 1300030901）

## 相关Issue

### Issue 1: 类目属性不完整
**问题**：导出的Excel上传到美团后报错，提示缺少必填属性（分辨率、屏幕比例、尺寸等）

**根本原因**：
1. 原始美团Excel中，某些商品的类目属性本身就不完整
2. 列映射不一致（"类目属性"映射到`categoryAttributes`）
3. 硬编码索引50可能读取错误的列

**解决方案**：
1. ✅ 统一列映射
2. ✅ 优化读取逻辑
3. ✅ 添加诊断日志
4. ⚠️ 需要在美团后台补全商品的类目属性（数据源问题）

### Issue 2: 无理由退货标签ID未存储
**问题**：导入时解析正确（日志显示1300030902），但数据库中为NULL，导出时使用默认值1300030895

**根本原因**：
- `ProductMapper.xml`的INSERT语句中缺少`no_reason_return_tag_id`字段
- 虽然Entity类有属性，解析逻辑也正确，但存储时被忽略

**解决方案**：
- ✅ 在INSERT语句中添加字段和参数绑定
- ✅ 确保完整的数据流：解析 → 存储 → 导出
