# 应用自动更新 - 测试指南

## 测试目标

验证应用自动更新功能是否正常工作，包括：
- 版本检查功能
- 更新提示显示
- 更新流程执行
- 缓存清除和页面刷新

## 测试前准备

### 1. 确认当前版本

当前代码中的版本号都是 `1.0.0`：

**前端版本**：
```javascript
// meituan-frontend/src/utils/updateChecker.js
const CURRENT_VERSION = '1.0.0'
```

**后端版本**：
```yaml
# meituan-backend/src/main/resources/application.yml
app:
  version: 1.0.0
```

## 测试方案

### 方案1：模拟有新版本（推荐）

这个方案最简单，只需要修改后端返回的版本号。

#### 步骤：

1. **修改后端版本号**

打开 `meituan-backend/src/main/resources/application.yml`，修改版本号：

```yaml
app:
  version: 1.0.1  # 改成 1.0.1（比前端的 1.0.0 高）
```

2. **修改更新日志（可选）**

打开 `meituan-backend/src/main/java/com/meituan/product/controller/VersionController.java`，修改更新内容：

```java
private String getUpdateLog() {
    return """
            ✨ 新功能
            - 测试自动更新功能
            - 这是一个测试版本
            
            🐛 问题修复
            - 修复测试问题
            
            🎨 界面优化
            - 优化测试体验
            """;
}
```

3. **重启后端服务**

```bash
# 停止后端
# 然后重新启动
cd meituan-backend
mvn spring-boot:run
```

4. **测试自动检查**

- 打开前端应用（或刷新页面）
- 等待 5 秒（应用启动后会自动检查）
- 应该会在右上角看到通知："发现新版本 v1.0.1 可用"

5. **测试手动检查**

- 进入"系统设置"页面
- 在右侧"应用信息"卡片中
- 点击"检查更新"按钮
- 应该会弹出更新对话框，显示版本号和更新内容

6. **测试更新流程**

- 在更新对话框中点击"立即更新"
- 会弹出确认对话框："更新将清除浏览器缓存并刷新页面，是否继续？"
- 点击"确定"
- 应该会显示"更新完成，即将重新加载应用..."
- 2秒后页面自动刷新

7. **验证更新后状态**

- 页面刷新后，打开浏览器控制台（F12）
- 查看日志，应该显示："开始检查更新，当前版本: 1.0.0"
- 再次点击"检查更新"，应该提示"已是最新版本"（因为后端是1.0.1，前端还是1.0.0）

### 方案2：完整版本更新测试

这个方案模拟真实的版本发布流程。

#### 步骤：

1. **修改前端版本号为 1.0.0**（确认当前是1.0.0）

```javascript
// meituan-frontend/src/utils/updateChecker.js
const CURRENT_VERSION = '1.0.0'
```

2. **修改后端版本号为 1.0.1**

```yaml
# meituan-backend/src/main/resources/application.yml
app:
  version: 1.0.1
```

3. **打包前端（当前版本 1.0.0）**

```bash
cd meituan-frontend
npm run build
```

4. **部署到服务器**

```bash
# 上传到服务器
scp -r dist/* user@server:/var/www/meituan-frontend/
```

5. **重启后端服务**

```bash
# 在服务器上重启后端
systemctl restart meituan-backend
```

6. **用户访问应用**

- 用户打开应用（版本1.0.0）
- 5秒后自动检查更新
- 发现服务器版本是1.0.1
- 显示更新提示

7. **用户点击更新**

- 清除缓存
- 刷新页面
- 加载新版本的文件

8. **修改前端版本号为 1.0.1**

```javascript
// meituan-frontend/src/utils/updateChecker.js
const CURRENT_VERSION = '1.0.1'
```

9. **重新打包并部署**

```bash
cd meituan-frontend
npm run build
scp -r dist/* user@server:/var/www/meituan-frontend/
```

10. **验证**

- 用户刷新页面后
- 再次检查更新
- 应该提示"已是最新版本"

## 调试技巧

### 查看控制台日志

打开浏览器控制台（F12），查看更新相关日志：

```
启动自动更新检查
开始检查更新，当前版本: 1.0.0
服务器最新版本: { version: '1.0.1', ... }
发现新版本: 1.0.1
```

### 查看网络请求

在浏览器开发者工具的 Network 标签中：

1. 找到 `/api/version/latest` 请求
2. 查看响应内容
3. 确认版本号是否正确

### 查看 localStorage

在浏览器控制台执行：

```javascript
// 查看最后检查时间
localStorage.getItem('last_update_check')

// 查看保存的版本号
localStorage.getItem('app_version')

// 清除检查时间（强制立即检查）
localStorage.removeItem('last_update_check')
```

### 手动触发检查

在浏览器控制台执行：

```javascript
// 手动触发更新检查（非静默）
import('@/utils/updateChecker').then(module => {
  module.default.checkUpdate(false)
})
```

## 常见问题

### Q1: 没有看到更新通知？

**检查项**：
1. 后端版本号是否比前端高？
2. 后端服务是否正常运行？
3. 网络请求是否成功？（查看 Network 标签）
4. 是否在5秒内刷新了页面？

### Q2: 点击更新后没有反应？

**检查项**：
1. 浏览器控制台是否有错误？
2. 是否阻止了页面刷新？
3. 是否有浏览器插件干扰？

### Q3: 更新后还是旧版本？

**可能原因**：
1. 浏览器缓存未清除 - 尝试硬刷新（Ctrl+F5）
2. 服务器文件未更新 - 检查服务器上的文件
3. CDN缓存 - 等待CDN缓存过期

### Q4: 如何回到测试前的状态？

```bash
# 1. 恢复前端版本号为 1.0.0
# 编辑 meituan-frontend/src/utils/updateChecker.js

# 2. 恢复后端版本号为 1.0.0
# 编辑 meituan-backend/src/main/resources/application.yml

# 3. 重启后端服务
```
