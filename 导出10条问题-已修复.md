# 导出10条问题 - 已修复

## 问题描述
导出全部商品时，只导出了10条数据，而不是全部数据。

## 根本原因
在 `ProductController.java` 中，`generateAllTemplate` 方法的 `limit` 参数设置了默认值 `defaultValue = "10"`：

```java
@RequestParam(value = "limit", required = false, defaultValue = "10") Integer limit
```

这导致即使前端没有传递 `limit` 参数，后端也会使用默认值10，从而只导出10条数据。

## 修复方案

### 修改前
```java
@PostMapping("/generate-all-template")
public ResponseEntity<byte[]> generateAllTemplate(
        @RequestParam(value = "merchantId", required = false) Long merchantId,
        @RequestParam(value = "limit", required = false, defaultValue = "10") Integer limit) {
    // ...
}
```

### 修改后
```java
@PostMapping("/generate-all-template")
public ResponseEntity<byte[]> generateAllTemplate(
        @RequestParam(value = "merchantId", required = false) Long merchantId,
        @RequestParam(value = "limit", required = false) Integer limit) {
    // ...
}
```

**关键变化**：移除了 `defaultValue = "10"`

## 逻辑说明

修改后的行为：
- 如果前端**不传递** `limit` 参数：`limit = null` → 导出**全部商品**
- 如果前端**传递** `limit = 10`：`limit = 10` → 导出**10条商品**（测试模式）

在 `ProductService.java` 中的判断逻辑：
```java
// 判断是否为测试模式（指定了limit）
boolean isTestMode = limit != null && limit > 0;

if (isTestMode) {
    // 测试模式：只查询指定数量的商品
    products = productMapper.selectRecent(merchantId, limit);
} else {
    // 正式模式：查询所有商品
    products = productMapper.selectByMerchantId(merchantId);
}
```

## 前端调用

前端代码正确，没有传递 `limit` 参数：

```javascript
export const generateAllTemplate = (merchantId = 1) => {
  return request.post('/products/generate-all-template', null, {
    params: { merchantId },  // 只传递 merchantId，不传递 limit
    responseType: 'blob',
    timeout: 1200000
  })
}
```
