# 应用自动更新 - 403错误修复

## 问题描述

点击"检查更新"时出现 403 错误：
```
http://127.0.0.1:8080/api/api/version/latest 403 (Forbidden)
```

## 问题原因

1. **URL重复问题**：`/api/api/version/latest` - API路径重复了
   - `baseURL` 已经包含了 `/api`
   - 请求时又加了 `/api`，导致路径错误

2. **权限问题**：版本检查接口被JWT拦截器拦截
   - `/api/version/latest` 需要登录才能访问
   - 但版本检查应该是公开接口，不需要登录

## 修复方案

### 1. 修复URL重复问题

**文件**：`meituan-frontend/src/utils/updateChecker.js`

**修改前**：
```javascript
const response = await request.get('/api/version/latest')
```

**修改后**：
```javascript
const response = await request.get('/version/latest')
```

**原因**：`request` 实例的 `baseURL` 已经是 `${API_BASE_URL}/api`，所以不需要再加 `/api`

### 2. 添加版本检查接口到白名单

**文件**：`meituan-backend/src/main/java/com/meituan/product/config/SecurityConfig.java`

**修改前**：
```java
.authorizeHttpRequests(auth -> auth
    .requestMatchers("/api/auth/**").permitAll()
    .requestMatchers("/api/company").permitAll()
    .requestMatchers("/api/app-versions/latest/**").permitAll()
    .requestMatchers("/api/app-versions/*/download").permitAll()
    .requestMatchers("/api/**").authenticated()
    .anyRequest().authenticated()
)
```

**修改后**：
```java
.authorizeHttpRequests(auth -> auth
    .requestMatchers("/api/auth/**").permitAll()
    .requestMatchers("/api/company").permitAll()
    .requestMatchers("/api/app-versions/latest/**").permitAll()
    .requestMatchers("/api/app-versions/*/download").permitAll()
    .requestMatchers("/api/version/**").permitAll()  // 新增这行
    .requestMatchers("/api/**").authenticated()
    .anyRequest().authenticated()
)
```

**原因**：版本检查接口应该是公开的，不需要登录就能访问

- 响应内容应该包含版本信息
