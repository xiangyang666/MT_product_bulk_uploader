# 类目属性缺失问题诊断

## 问题描述
导出的Excel上传到美团后报错：
```
【信息错误】属性[分辨率]不能为空；属性[屏幕比例]不能为空；属性[能效等级]不能为空；
属性[接口]不能为空；属性[屏幕刷新率（Hz）]不能为空；属性[面板]不能为空；
属性[曲率]不能为空；，属性【尺寸（英寸）】必填，请填写属性信息
```

## 根本原因
美团平台对每个商品类目都有**必填属性要求**。显示器类商品必须包含：
- 分辨率
- 屏幕比例  
- 尺寸（英寸）
- 能效等级
- 接口
- 屏幕刷新率（Hz）
- 面板
- 曲率

## 诊断步骤

### 1. 检查原始美团Excel
打开从美团后台导出的原始Excel文件，找到"类目属性"列（通常是第51列，索引50），检查：
- ✅ 该列是否存在
- ✅ 列名是否为"类目属性"或"商品属性"
- ✅ 内容格式是否为：`品牌：xxx。分辨率：xxx。屏幕比例：xxx。...`

### 2. 检查导入日志
查看后端日志 `meituan-backend/springboot-*.log`，搜索"类目属性"：
```
grep "类目属性" meituan-backend/springboot-*.log
```

应该看到类似：
```
第3行：类目属性 = 品牌：三星（SAMSUNG）。分辨率：5120*1440。屏幕比例：32:9。尺寸（英寸）：49。能效等级：1级。接口：hdmi。屏幕刷新率（Hz）：240。曲率：3000R。
```

**如果日志中类目属性为空或不完整，说明导入时读取失败。**

### 3. 检查数据库存储
执行SQL查询：
```sql
SELECT 
    id,
    LEFT(product_name, 30) as product_name,
    category_name,
    LENGTH(product_attributes) as attr_length,
    product_attributes
FROM t_product
WHERE merchant_id = 1
  AND category_name LIKE '%显示器%'
ORDER BY id DESC
LIMIT 5;
```

**如果 `product_attributes` 为 NULL 或长度很短，说明存储时丢失了数据。**

### 4. 检查导出的Excel
打开从平台导出的Excel文件，找到"类目属性"列，检查：
- ✅ 该列的值是否完整
- ✅ 是否包含所有必填属性

## 可能的问题和解决方案

### 问题1：列索引不匹配
**症状**：日志显示"类目属性 = null"或为空

**原因**：代码中硬编码了索引50，但实际Excel中"类目属性"列不在第51列

**解决方案**：
修改 `MeituanFormatParser.java`，不要硬编码索引，完全依赖列映射：

```java
// 修改前（硬编码索引50）
String productAttrs = extractFieldByIndex(row, 50);

// 修改后（完全依赖映射）
String productAttrs = extractField(row, "productAttributes", columnMapping);
if (productAttrs == null || productAttrs.trim().isEmpty()) {
    productAttrs = extractField(row, "categoryAttributes", columnMapping);
}
```

### 问题2：列名不匹配
**症状**：日志显示"类目属性 = null"，但Excel中该列确实存在

**原因**：Excel中的列名可能是"商品属性"而不是"类目属性"

**解决方案**：
检查 `FormatDetector.java` 中的列映射规则，确保包含所有可能的列名变体。

### 问题3：数据被截断
**症状**：日志显示类目属性完整，但数据库中被截断

**原因**：字段长度限制（不太可能，因为是TEXT类型）

**解决方案**：
检查数据库字段定义：
```sql
SHOW CREATE TABLE t_product;
```
确保 `product_attributes` 是 `TEXT` 或 `LONGTEXT` 类型。

### 问题4：原始Excel数据不完整
**症状**：日志和数据库都显示类目属性不完整

**原因**：从美团导出的原始Excel本身就缺少必填属性

**解决方案**：
这是美团后台的数据问题，需要：
1. 在美团后台手动补全商品的类目属性
2. 重新导出Excel
3. 再导入到本平台

## 快速验证方法

### 方法1：对比原始Excel和导出Excel
1. 打开美团导出的原始Excel
2. 找到一个显示器商品，复制"类目属性"列的值
3. 导入到平台
4. 从平台导出Excel
5. 对比两个Excel中"类目属性"列的值是否一致

**如果不一致，说明导入或导出过程中丢失了数据。**

### 方法2：手动测试
1. 创建一个测试Excel，只包含1个显示器商品
2. 在"类目属性"列填写完整的属性：
   ```
   品牌：测试品牌。分辨率：1920*1080。屏幕比例：16:9。尺寸（英寸）：27。能效等级：1级。接口：HDMI。屏幕刷新率（Hz）：60。面板：IPS。曲率：平面。
   ```
3. 导入到平台
4. 查看日志，确认类目属性被正确读取
5. 查询数据库，确认类目属性被正确存储
6. 导出Excel，确认类目属性被正确导出
7. 上传到美团，看是否还报错

## 临时解决方案

如果问题紧急，可以：
1. 直接在美团后台编辑商品，补全必填属性
2. 或者在本平台导出后，手动在Excel中补全"类目属性"列
3. 然后再上传到美团
