# 商品图片上传功能 - 完整实现说明

## 功能概述

在商品管理页面添加了完整的图片上传和导出功能：

1. **图片上传**：每个商品可上传最多5张图片，使用阿里云OSS存储
2. **图片管理**：查看、删除商品图片
3. **图片导出**：按美团规范导出商品图片为ZIP压缩包

## 部署步骤

### 1. 数据库迁移

执行SQL脚本添加 `product_images` 字段：

```bash
mysql -u root -p < database-migration-product-images.sql
```

或手动执行：

```sql
USE meituan_product;

ALTER TABLE t_product 
ADD COLUMN product_images TEXT COMMENT '商品图片URL列表（JSON数组，最多5张）' 
AFTER image_url;
```

### 2. 后端配置

#### 2.1 添加Maven依赖

在 `meituan-backend/pom.xml` 中添加：

```xml
<!-- 图片处理库 -->
<dependency>
    <groupId>net.coobird</groupId>
    <artifactId>thumbnailator</artifactId>
    <version>0.4.19</version>
</dependency>

<!-- 阿里云OSS SDK -->
<dependency>
    <groupId>com.aliyun.oss</groupId>
    <artifactId>aliyun-sdk-oss</artifactId>
    <version>3.17.1</version>
</dependency>
```

#### 2.2 配置阿里云OSS

在 `meituan-backend/src/main/resources/application.yml` 中添加：

```yaml
# 阿里云OSS配置
aliyun:
  oss:
    endpoint: oss-cn-hangzhou.aliyuncs.com  # 替换为你的OSS地域
    access-key-id: YOUR_ACCESS_KEY_ID       # 替换为你的AccessKeyId
    access-key-secret: YOUR_ACCESS_KEY_SECRET  # 替换为你的AccessKeySecret
    bucket-name: your-bucket-name           # 替换为你的Bucket名称
    base-url: https://your-bucket-name.oss-cn-hangzhou.aliyuncs.com  # 访问URL前缀
```

**获取OSS配置信息**：
1. 登录阿里云控制台：https://oss.console.aliyun.com/
2. 创建Bucket（如果还没有）
   - 选择地域（如：华东1-杭州）
   - 设置读写权限为"公共读"
3. 获取AccessKey
   - 进入用户中心 → AccessKey管理
   - 创建AccessKey（记录AccessKeyId和AccessKeySecret）
4. 记录配置信息
   - Endpoint：如 `oss-cn-hangzhou.aliyuncs.com`
   - Bucket名称：如 `meituan-products`
   - Base URL：如 `https://meituan-products.oss-cn-hangzhou.aliyuncs.com`

#### 2.3 重新编译后端

```bash
cd meituan-backend
mvn clean package -DskipTests
```

#### 2.4 重启后端服务

停止现有服务，启动新编译的jar文件。

### 3. 前端部署

#### 3.1 重新编译前端

```bash
cd meituan-frontend
npm run build
```

#### 3.2 部署前端

将 `dist` 目录部署到服务器。

## 功能使用说明

### 1. 上传商品图片

1. 进入"商品管理"页面
2. 找到要上传图片的商品
3. 点击"管理图片"按钮
4. 在弹出的对话框中点击"上传图片"
5. 选择图片文件（支持jpg、png、jpeg等格式）
6. 等待上传完成

**限制**：
- 每个商品最多5张图片
- 每张图片不超过10MB
- 只支持图片格式

### 2. 删除商品图片

1. 在图片管理对话框中
2. 点击图片下方的"删除"按钮
3. 确认删除

### 3. 导出商品图片

1. 在商品管理页面点击"导出商品图片"按钮
2. 选择图片命名方式：
   - 商品名称：使用商品名称命名（如：苹果-1.jpg）
   - 条形码：使用条形码命名（如：6901234567890-1.jpg）
   - 店内码/货号：使用店内码命名（如：SKU001-1.jpg）
3. 等待导出完成
4. 自动下载ZIP压缩包

**导出规范**（按美团要求）：
- 图片格式：自动转换为JPG
- 图片大小：超过5M自动压缩
- 命名规则：`商品标识-序号.jpg`（如：苹果-1.jpg、苹果-2.jpg）
- 特殊字符处理：`/` → `_`，`*` → `^`
- 最多导出300张图片

## API接口说明

### 上传商品图片

```
POST /api/products/{productId}/images/upload
Content-Type: multipart/form-data

参数：
- file: 图片文件

返回：
{
  "code": 200,
  "message": "上传成功",
  "data": {
    "imageUrl": "https://xxx.oss-cn-hangzhou.aliyuncs.com/products/images/xxx.jpg",
    "imageUrls": ["url1", "url2", ...]
  }
}
```

### 删除商品图片

```
DELETE /api/products/{productId}/images?imageUrl=xxx

返回：
{
  "code": 200,
  "message": "删除成功",
  "data": {
    "imageUrls": ["url1", "url2", ...]
  }
}
```

### 获取商品图片列表

```
GET /api/products/{productId}/images

返回：
{
  "code": 200,
  "message": "获取成功",
  "data": {
    "imageUrls": ["url1", "url2", ...]
  }
}
```

### 导出商品图片

```
POST /api/products/export-images
Content-Type: application/json

请求体：
{
  "merchantId": 1,
  "namingType": "PRODUCT_NAME",  // PRODUCT_NAME | BARCODE | STORE_CODE
  "productIds": [1, 2, 3]  // 可选，不传则导出全部
}

返回：ZIP文件流
```

## 故障排查

### 问题1：OSS初始化失败

**症状**：日志显示"阿里云OSS配置未设置"

**检查**：
1. `application.yml` 中的OSS配置是否正确
2. AccessKey是否有效（在阿里云控制台验证）
3. Bucket是否存在
4. 网络是否可以访问OSS

**解决**：
- 重新配置 `application.yml`
- 确保AccessKey有OSS读写权限
- 检查防火墙设置

### 问题2：图片上传失败

**症状**：上传时报错"上传失败"

**检查**：
1. 文件大小是否超过10MB
2. 文件格式是否为图片
3. OSS Bucket权限是否正确
4. 后端日志中的详细错误信息

**解决**：
- 压缩图片后重试
- 设置Bucket为"公共读"权限
- 检查OSS账户余额

### 问题3：图片无法访问

**症状**：图片上传成功但无法显示

**检查**：
1. Bucket读写权限是否为"公共读"
2. 图片URL是否正确
3. 浏览器控制台是否有CORS错误

**解决**：
- 在OSS控制台设置Bucket为"公共读"
- 配置CORS规则（允许所有来源）

### 问题4：导出超时

**症状**：导出时提示超时

**原因**：图片数量过多

**解决**：
- 分批导出（选择部分商品）
- 增加超时时间（前端API中的timeout参数）
- 优化网络连接

## 技术细节

### 图片存储格式

商品图片以JSON数组格式存储在 `product_images` 字段：

```json
[
  "https://xxx.oss-cn-hangzhou.aliyuncs.com/products/images/abc123.jpg",
  "https://xxx.oss-cn-hangzhou.aliyuncs.com/products/images/def456.jpg"
]
```

### 图片压缩策略

导出时，如果图片超过5MB，使用Thumbnailator库进行压缩：

1. 第一次压缩：质量85%
2. 如果仍超过5MB：质量70%
3. 保持原尺寸不变

### 文件命名规则

按美团规范处理文件名：

1. 基础名称：商品名称/条形码/店内码
2. 序号：从1开始（如：-1、-2、-3）
3. 扩展名：统一为 `.jpg`
4. 特殊字符：`/` → `_`，`*` → `^`

示例：
- 商品名称："苹果/红富士" → "苹果_红富士-1.jpg"
- 条形码："6901234567890" → "6901234567890-1.jpg"
