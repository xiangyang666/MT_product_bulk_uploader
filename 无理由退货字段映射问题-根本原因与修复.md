# 无理由退货字段映射问题 - 根本原因与修复方案

## 问题现象

导入 "7天无理由退货（一次性包装破损不支持）" 后，导出显示为 "1300030895"（不支持7天无理由退货），而非正确的 "1300030902"。

## 根本原因分析

### 1. **数据库字段缺失** ❌

**问题**：`t_product` 表中缺少 `no_reason_return_tag_id` 字段

```sql
-- 实际表结构（错误）
no_reason_return  tinyint  -- 只有布尔字段

-- 应该的表结构（正确）
no_reason_return        tinyint
no_reason_return_tag_id varchar(20)  -- 标签ID字段（缺失！）
```

**影响**：
- 代码解析时正确识别了标签ID（"1300030902"）
- 但保存到数据库时，`no_reason_return_tag_id` 字段不存在，被忽略
- 只保存了 `no_reason_return=0`（布尔值）
- 导出时读取到 `no_reason_return=0`，映射为 "1300030895"

### 2. **映射规则配置错误** ❌

**问题**：`mapping-rules.json` 中"无理由退货"映射到了错误的字段

```json
// 实际配置（错误）
{
  "noReasonReturn": {
    "fieldName": "noReasonReturn",  // ❌ 布尔字段
    "displayName": "无理由退货",
    ...
  }
}

// 应该配置（正确）
{
  "noReasonReturnTagId": {
    "fieldName": "noReasonReturnTagId",  // ✅ 标签ID字段
    "displayName": "无理由退货",
    ...
  }
}
```

**影响**：
- 自适应映射引擎（AdaptiveMappingEngine）根据配置将"无理由退货"列映射到 `noReasonReturn`
- 而非代码中预期的 `noReasonReturnTagId`
- 导致标签ID无法正确保存

### 3. **代码逻辑问题** ⚠️

**问题**：`MeituanFormatParser.java:140` 中的兼容逻辑

```java
// 问题代码
product.setNoReasonReturn("1300030901".equals(tagId) ? 1 : 0);
```

**影响**：
- 只有 "1300030901"（通用7天无理由退货）才会设置 `no_reason_return=1`
- 其他标签（如 1300030902、1300030903 等）都会设置 `no_reason_return=0`
- 如果 `no_reason_return_tag_id` 字段不存在，导出时会使用 `no_reason_return=0`，显示 "1300030895"

## 已完成的修复

### ✅ 1. 添加数据库字段

```sql
ALTER TABLE t_product
ADD COLUMN no_reason_return_tag_id VARCHAR(20) DEFAULT NULL
COMMENT '无理由退货标签ID' AFTER no_reason_return;
```

### ✅ 2. 修复映射规则配置

**文件**：`meituan-backend/src/main/resources/mapping-rules.json`

```diff
- "noReasonReturn": {
-   "fieldName": "noReasonReturn",
+ "noReasonReturnTagId": {
+   "fieldName": "noReasonReturnTagId",
    "displayName": "无理由退货",
    ...
  }
```

### ✅ 3. 改进标签匹配逻辑

**文件**：`meituan-backend/src/main/java/com/meituan/product/service/MeituanFormatParser.java`

- 使用 `contains()` 方式匹配，更宽松
- 添加详细日志（log.info）
- 优先匹配更具体的标签

```java
// 新逻辑
if (trimmedValue.contains("一次性包装破损") && trimmedValue.contains("不支持")) {
    log.info("匹配到: 7天无理由退货（一次性包装破损不支持）-> 1300030902");
    return "1300030902";
}
```

## 故障排查

### 问题1：导出仍显示 1300030895

**可能原因**：
1. 数据库字段未添加 ❌
2. 旧数据未清理 ❌
3. 应用未重启 ❌

**解决方案**：
```bash
# 检查字段
mysql -h 106.55.102.48 -u root -p meituan_product -e "DESCRIBE t_product;" | grep no_reason_return_tag_id

# 清理数据
mysql -h 106.55.102.48 -u root -p meituan_product -e "DELETE FROM t_product WHERE merchant_id = 1;"

# 重启应用
ps aux | grep meituan-product | awk '{print $2}' | xargs kill -9
java -jar meituan-backend/target/meituan-product-1.0.0.jar
```

### 问题2：日志显示"未匹配到任何标签"

**可能原因**：Excel文本包含特殊字符

**解决方案**：
1. 从美团平台复制粘贴标签名称
2. 使用记事本清理Excel中的文本
3. 查看日志中打印的字符详情

**可能原因**：Maven依赖问题

**解决方案**：
```bash
cd meituan-backend
mvn clean install -DskipTests
```
