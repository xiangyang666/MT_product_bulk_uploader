# 美团模板导出格式修复说明

## 问题描述

导出的美团模板数据格式不符合美团平台要求，导致上传时出现以下错误：

1. **无理由退货字段**：需要填写标签ID，不是 0/1 或"是/否"
2. **售卖状态字段**：值不是 0 或 1（可能输出了"上架/下架"等文本）
3. **可售时间字段**：时间格式错误，需要支持多种格式

## 最新修复内容（2026-03-06）

### 1. 无理由退货字段修复（完整版）

**问题**: 美团要求填写标签ID，导入时可能是文本描述，导出时必须是标签ID

**解决方案**:
1. 添加新字段 `no_reason_return_tag_id` 存储标签ID
2. 导入时，自动将文本描述转换为标签ID
3. 导出时，优先使用 `no_reason_return_tag_id`，如果为空则根据 `no_reason_return` 映射

**支持的导入格式**:
- 标签ID：`1300030895`、`1300030901` 等
- 文本描述：`不支持7天无理由退货`、`7天无理由退货` 等
- 布尔值：`0`/`1`、`是`/`否`、`true`/`false`

**标签ID映射规则**:
| 输入值 | 标签ID | 标签名称 |
|--------|--------|----------|
| 不支持7天无理由退货、0、否、false | 1300030895 | 不支持7天无理由退货 |
| 7天无理由退货（一次性包装破损不支持）| 1300030902 | 7天无理由退货（一次性包装破损不支持）|
| 7天无理由退货（激活后不支持）| 1300030903 | 7天无理由退货（激活后不支持）|
| 7天无理由退货（使用后不支持）| 1300030904 | 7天无理由退货（使用后不支持）|
| 7天无理由退货（安装后不支持）| 1300030905 | 7天无理由退货（安装后不支持）|
| 7天无理由退货（定制类不支持）| 1300030906 | 7天无理由退货（定制类不支持）|
| 7天无理由退货、七天无理由退货、1、是、true | 1300030901 | 7天无理由退货 |
| 已经是标签ID格式（13开头10位数字）| 原值 | 直接使用 |

**代码实现**:
```java
// MeituanFormatParser.java - 导入时解析
private String parseNoReasonReturnTagId(String value) {
    if (value == null || value.trim().isEmpty()) {
        return "1300030895"; // 默认：不支持
    }
    
    String trimmedValue = value.trim();
    
    // 如果已经是标签ID格式，直接返回
    if (trimmedValue.matches("^13\\d{8}$")) {
        return trimmedValue;
    }
    
    // 根据文本描述映射
    if (trimmedValue.contains("不支持") || trimmedValue.equals("0")) {
        return "1300030895";
    } else if (trimmedValue.contains("一次性包装破损不支持")) {
        return "1300030902";
    } else if (trimmedValue.contains("激活后不支持")) {
        return "1300030903";
    } else if (trimmedValue.contains("使用后不支持")) {
        return "1300030904";
    } else if (trimmedValue.contains("安装后不支持")) {
        return "1300030905";
    } else if (trimmedValue.contains("定制类不支持")) {
        return "1300030906";
    } else if (trimmedValue.contains("7天") || trimmedValue.equals("1")) {
        return "1300030901";
    }
    
    return "1300030895"; // 默认
}

// ExcelService.java - 导出时使用
String tagId = product.getNoReasonReturnTagId();
if (tagId == null || tagId.trim().isEmpty()) {
    // 向后兼容：根据旧字段映射
    tagId = (product.getNoReasonReturn() == 1) ? "1300030901" : "1300030895";
}
cell.setCellValue(tagId);
```

### 2. 可售时间字段修复

**问题**: 美团支持多种可售时间格式，之前只支持简单的时间段

**支持的格式**:
1. **指定时间点开售**: `HH:mm`（如 `09:00`）
2. **指定时间段售卖**: `HH:mm-HH:mm`（如 `09:00-22:00`）
3. **营业全时段可售**: `营业全时段可售`
4. **周期性可售**: `时间段1，时间段2;周一，周二`（用中文分号隔开）
5. **不填写**: 默认为`营业全时段可售`

**修复逻辑**:
```java
String timeValue = "营业全时段可售"; // 默认值

if (product.getAvailableTime() != null && !product.getAvailableTime().trim().isEmpty()) {
    String time = product.getAvailableTime().trim();
    
    // 支持"全天"或"营业全时段可售"
    if ("全天".equals(time) || "营业全时段可售".equals(time)) {
        timeValue = "营业全时段可售";
    }
    // 支持时间点：HH:mm
    else if (time.matches("^(0?[0-9]|1[0-9]|2[0-3]):([0-5][0-9])$")) {
        timeValue = time;
    }
    // 支持时间段：HH:mm-HH:mm
    else if (time.matches("^(0?[0-9]|1[0-9]|2[0-3]):([0-5][0-9])-(0?[0-9]|1[0-9]|2[0-3]):([0-5][0-9])$")) {
        timeValue = time;
    }
    // 支持周期性可售：包含中文分号
    else if (time.contains("；") || time.contains(";")) {
        timeValue = time;
    }
}
cell.setCellValue(timeValue);
```

**周期性可售示例**:
- `09:00-12:00，14:00-18:00；周一，周三，周五`
- `10:00-22:00；周一，周二，周三，周四，周五`

### 1. 售卖状态字段修复

**修改位置**: `ExcelService.java` - `fillCellByHeaderName()` 方法

**修复逻辑**:
- 如果数据库中存储的是 "0" 或 "1"，直接使用
- 如果是 "上架"、"在售"、"销售中"，转换为 "1"
- 如果是 "下架"、"停售"、"已下架"，转换为 "0"
- 其他情况默认为 "1"（上架）

```java
if (product.getSaleStatus() != null) {
    String status = product.getSaleStatus().trim();
    if ("0".equals(status) || "1".equals(status)) {
        cell.setCellValue(status);
    } else if ("上架".equals(status) || "在售".equals(status) || "销售中".equals(status)) {
        cell.setCellValue("1");
    } else if ("下架".equals(status) || "停售".equals(status) || "已下架".equals(status)) {
        cell.setCellValue("0");
    } else {
        cell.setCellValue("1"); // 默认上架
    }
} else {
    cell.setCellValue("1"); // 默认上架
}
```

### 2. 无理由退货字段修复

**修改位置**: `ExcelService.java` - `fillCellByHeaderName()` 方法

**修复逻辑**:
- 直接输出数据库中的 Integer 值（0 或 1）
- 不再转换为"是/否"文本
- 默认值为 "0"（不支持无理由退货）

```java
if (product.getNoReasonReturn() != null) {
    cell.setCellValue(product.getNoReasonReturn().toString());
} else {
    cell.setCellValue("0");
}
```

### 3. 可售时间字段修复

**修改位置**: `ExcelService.java` - `fillCellByHeaderName()` 方法

**修复逻辑**:
- 验证时间格式是否符合 `HH:mm-HH:mm` 格式
- 支持 "全天" 或 "00:00-23:59" 格式
- 格式不正确时，默认设置为 "00:00-23:59"（全天）

```java
if (product.getAvailableTime() != null && !product.getAvailableTime().trim().isEmpty()) {
    String time = product.getAvailableTime().trim();
    if (time.matches("\\d{1,2}:\\d{2}-\\d{1,2}:\\d{2}") || 
        "全天".equals(time) || 
        "00:00-23:59".equals(time)) {
        cell.setCellValue(time);
    } else {
        cell.setCellValue("00:00-23:59"); // 格式不正确，设置为全天
    }
} else {
    cell.setCellValue("00:00-23:59"); // 默认全天
}
```

首先执行数据库迁移脚本，添加新字段：

```bash

```

或者手动执行：
```sql
-- 添加无理由退货标签ID字段
ALTER TABLE t_product ADD COLUMN IF NOT EXISTS no_reason_return_tag_id VARCHAR(20) DEFAULT NULL COMMENT '无理由退货标签ID' AFTER no_reason_return;

-- 将现有数据转换为标签ID
UPDATE t_product
SET no_reason_return_tag_id = CASE
    WHEN no_reason_return = 1 THEN '1300030901'
    ELSE '1300030895'
END
WHERE no_reason_return_tag_id IS NULL;
```
