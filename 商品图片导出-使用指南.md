# 商品图片批量导出 - 使用指南

## 功能概述

在商品管理页面批量导出商品图片，按照美团规范自动处理图片命名和格式。

## 使用步骤

### 1. 准备工作

#### 后端配置

1. **添加Maven依赖**

在 `meituan-backend/pom.xml` 中添加图片处理库：

```xml
<!-- 图片处理库 Thumbnailator -->
<dependency>
    <groupId>net.coobird</groupId>
    <artifactId>thumbnailator</artifactId>
    <version>0.4.19</version>
</dependency>
```

2. **重新编译后端**

```bash
cd meituan-backend
mvn clean package -DskipTests
```

3. **重启后端服务**

#### 前端配置

前端代码已经准备好，无需额外配置。

### 2. 使用功能

#### 2.1 在商品管理页面

1. 进入"商品管理"页面
2. 点击"导出商品图片"按钮
3. 选择命名方式：
   - **商品名称**：使用商品名称命名（推荐）
   - **条形码**：使用UPC/EAN条形码命名
   - **店内码/货号**：使用店内码命名

4. 选择导出范围：
   - **导出全部**：导出所有商品的图片
   - **导出选中**：只导出勾选的商品图片

5. 点击"确认导出"
6. 等待处理完成
7. 自动下载ZIP文件

#### 2.2 导出结果

ZIP文件包含：
- 所有商品图片
- 按规范命名
- 自动压缩超过5M的图片
- 所有图片在同一个文件夹（无嵌套）

## 图片命名规则

### 命名格式

```
{基础名称}-{序号}.jpg
```

### 示例

#### 使用商品名称
```
Apple iPhone 15 Pro-1.jpg
Apple iPhone 15 Pro-2.jpg
小米14 Ultra-1.jpg
```

#### 使用条形码
```
6901234567890-1.jpg
6901234567890-2.jpg
```

#### 使用店内码
```
MT001-1.jpg
MT001-2.jpg
```

### 特殊字符处理

Windows不支持的字符会自动替换：
- `/` → `_`
- `*` → `^`
- 其他非法字符会被移除

**示例**：
```
原始名称：Apple iPhone 15 Pro/Max*
处理后：Apple_iPhone_15_Pro_Max^-1.jpg
```

## 图片处理规范

### 自动处理

1. **格式转换**：所有图片统一转为JPG格式
2. **大小压缩**：超过5M的图片自动压缩
3. **质量保持**：压缩质量设置为85%，保持清晰度

### 限制说明

- **单次最多**：300张图片
- **文件夹最大**：1G
- **单张图片**：小于5M（自动压缩）

如果商品图片超过300张，系统会提示分批导出。

## 注意事项

### 1. 图片来源

- 图片必须已上传到OSS
- 确保图片URL可访问
- 建议使用公网可访问的OSS地址

### 2. 命名选择

- **商品名称**：最直观，推荐使用
- **条形码**：适合有标准条形码的商品
- **店内码**：适合内部管理使用

### 3. 导出时间

- 图片数量越多，导出时间越长
- 100张图片约需1-2分钟
- 请耐心等待，不要关闭页面

### 4. 网络要求

- 需要稳定的网络连接
- 从OSS下载图片需要时间
- 建议在网络良好时导出

## 常见问题

### Q1: 导出失败怎么办？

**可能原因**：
- 网络不稳定
- OSS图片无法访问
- 图片数量过多

**解决方法**：
1. 检查网络连接
2. 验证OSS配置
3. 分批导出（选择部分商品）

### Q2: 图片命名重复怎么办？

如果多个商品使用相同的命名基础（如相同的商品名称），后面的图片会覆盖前面的。

**解决方法**：
- 使用条形码或店内码命名（更唯一）
- 或者手动修改商品名称，确保唯一性

### Q3: 压缩后图片不清晰？

系统使用85%的压缩质量，通常不会影响清晰度。

**如果需要更高质量**：
- 在上传时就控制图片大小在5M以内
- 或者联系管理员调整压缩参数

### Q4: 导出的ZIP文件在哪里？

ZIP文件会自动下载到浏览器的默认下载目录。

**Windows默认**：`C:\Users\{用户名}\Downloads\`
