# 检查并修复服务器时间问题

## 问题描述

用户注册时间显示为 `2026-02-23T06:18:08`，年份错误（显示未来时间）。

## 原因分析

后端使用 `LocalDateTime.now()` 获取当前时间，这个时间来自服务器的系统时间。
如果显示 2026 年，说明服务器的系统时间设置错误。

## 解决方案

### 方法 1：检查服务器时间

在服务器上执行：

```bash
# 查看当前系统时间
date

# 查看时区设置
timedatectl

# 或
date +"%Y-%m-%d %H:%M:%S %Z"
```

### 方法 2：修正服务器时间

#### 如果时间错误，需要修正：

```bash
# 1. 设置正确的时区（中国时区）
sudo timedatectl set-timezone Asia/Shanghai

# 2. 启用 NTP 时间同步
sudo timedatectl set-ntp true

# 3. 手动同步时间（如果 NTP 不可用）
sudo ntpdate ntp.aliyun.com
# 或
sudo ntpdate time.windows.com

# 4. 如果没有 ntpdate，安装它
sudo apt-get install ntpdate

# 5. 验证时间是否正确
date
```

#### 如果需要手动设置时间：

```bash
# 格式：sudo date -s "YYYY-MM-DD HH:MM:SS"
sudo date -s "2025-02-23 14:30:00"

# 然后同步硬件时钟
sudo hwclock --systohc
```

### 方法 3：修复数据库中的错误时间

如果服务器时间已经修正，但数据库中已有错误的时间数据，需要更新：

```sql
-- 1. 查看当前错误的时间数据
SELECT id, username, created_at, updated_at 
FROM users 
WHERE YEAR(created_at) > 2025;

-- 2. 修正用户表的创建时间（将 2026 年改为 2025 年）
UPDATE users 
SET created_at = DATE_SUB(created_at, INTERVAL 1 YEAR),
    updated_at = DATE_SUB(updated_at, INTERVAL 1 YEAR)
WHERE YEAR(created_at) > 2025;

-- 3. 修正操作日志表的时间
UPDATE operation_logs 
SET created_at = DATE_SUB(created_at, INTERVAL 1 YEAR)
WHERE YEAR(created_at) > 2025;

-- 4. 修正生成文件表的时间
UPDATE generated_files 
SET created_at = DATE_SUB(created_at, INTERVAL 1 YEAR),
    expires_at = DATE_SUB(expires_at, INTERVAL 1 YEAR)
WHERE YEAR(created_at) > 2025;

-- 5. 修正应用版本表的时间
UPDATE app_versions 
SET created_at = DATE_SUB(created_at, INTERVAL 1 YEAR),
    updated_at = DATE_SUB(updated_at, INTERVAL 1 YEAR)
WHERE YEAR(created_at) > 2025;

-- 6. 验证修正结果
SELECT id, username, created_at, updated_at 
FROM users 
ORDER BY created_at DESC 
LIMIT 10;
```

### 方法 4：前端临时修复（不推荐，治标不治本）

如果暂时无法修复服务器时间，可以在前端做临时处理：

修改 `Settings.vue` 和 `ProfileSettings.vue` 中的 `formatDate` 函数：

```javascript
// 格式化日期
const formatDate = (dateString) => {
  if (!dateString) return ''
  
  let date = new Date(dateString)
  
  // 临时修复：如果年份大于当前年份，减去1年
  const now = new Date()
  if (date.getFullYear() > now.getFullYear()) {
    date = new Date(date.getTime() - 365 * 24 * 60 * 60 * 1000)
  }
  
  const currentYear = now.getFullYear()
  const dateYear = date.getFullYear()
  
  const month = String(date.getMonth() + 1).padStart(2, '0')
  const day = String(date.getDate()).padStart(2, '0')
  
  if (dateYear === currentYear) {
    return `${month}-${day}`
  }
  
  return `${dateYear}-${month}-${day}`
}
```

## 预防措施

### 1. 启用 NTP 时间同步

```bash
# 安装 NTP
sudo apt-get install ntp

# 启用并启动 NTP 服务
sudo systemctl enable ntp
sudo systemctl start ntp

# 验证 NTP 同步状态
ntpq -p
```

### 2. 配置时间同步服务器

编辑 `/etc/ntp.conf`：

```bash
sudo vi /etc/ntp.conf

# 添加或修改以下行
server ntp.aliyun.com iburst
server time.windows.com iburst
server cn.pool.ntp.org iburst
```

### 3. 定期检查时间

创建定时任务检查时间：

```bash
# 编辑 crontab
crontab -e

# 添加每天检查时间的任务
0 2 * * * /usr/sbin/ntpdate ntp.aliyun.com >> /var/log/ntp-sync.log 2>&1
```

## 常见问题

### Q1: ntpdate 命令不存在

```bash
# Ubuntu/Debian
sudo apt-get install ntpdate

# CentOS/RHEL
sudo yum install ntpdate
```

### Q2: 时间同步失败

```bash
# 检查防火墙是否阻止 NTP 端口（123）
sudo ufw status
sudo ufw allow 123/udp

# 或临时关闭防火墙测试
sudo ufw disable
```

### Q3: 修改时间后服务异常

```bash
# 重启所有相关服务
sudo systemctl restart mysql
sudo systemctl restart nginx
# 重启你的 Java 应用
```

## 注意事项

1. **修改系统时间前先备份数据库**
2. **在业务低峰期进行修改**
3. **修改后需要重启所有相关服务**
4. **确保时区设置正确**（中国使用 Asia/Shanghai）
5. **启用 NTP 自动同步，避免时间再次漂移**
