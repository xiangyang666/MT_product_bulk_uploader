# 无理由退货字段修复 - 快速部署指南

## 问题描述

美团平台报错：
```
检测到Excel表格中，无理由退货字段「无理由退货标签id格式有误或不在当前商品类目可选范围内」无法识别
```

## 根本原因

1. 导入时，用户填写的是文本描述（如"不支持7天无理由退货"）
2. 导出时，需要转换为美团平台的标签ID（如"1300030895"）
3. 之前的实现只支持简单的 0/1 映射，不支持文本描述

## 解决方案

添加新字段 `no_reason_return_tag_id` 存储标签ID，并实现智能转换逻辑。

## 部署步骤

### 步骤1：执行数据库迁移

```bash
# 方式1：使用脚本
mysql -u root -p meituan_product < database-migration-no-reason-return-tag.sql

# 方式2：手动执行SQL
mysql -u root -p
```

```sql
USE meituan_product;

-- 添加新字段
ALTER TABLE t_product ADD COLUMN IF NOT EXISTS no_reason_return_tag_id VARCHAR(20) DEFAULT NULL COMMENT '无理由退货标签ID' AFTER no_reason_return;

-- 转换现有数据
UPDATE t_product
SET no_reason_return_tag_id = CASE
    WHEN no_reason_return = 1 THEN '1300030901'
    ELSE '1300030895'
END
WHERE no_reason_return_tag_id IS NULL;

-- 验证结果
SELECT 
    no_reason_return,
    no_reason_return_tag_id,
    COUNT(*) as count
FROM t_product
GROUP BY no_reason_return, no_reason_return_tag_id;
```

## 导入格式支持

系统会自动将以下格式转换为标签ID：

| 输入值 | 转换为标签ID | 标签名称 |
|--------|-------------|----------|
| 不支持7天无理由退货 | 1300030895 | 不支持7天无理由退货 |
| 0、否、false | 1300030895 | 不支持7天无理由退货 |
| 7天无理由退货（一次性包装破损不支持）| 1300030902 | 7天无理由退货（一次性包装破损不支持）|
| 7天无理由退货（激活后不支持）| 1300030903 | 7天无理由退货（激活后不支持）|
| 7天无理由退货（使用后不支持）| 1300030904 | 7天无理由退货（使用后不支持）|
| 7天无理由退货（安装后不支持）| 1300030905 | 7天无理由退货（安装后不支持）|
| 7天无理由退货（定制类不支持）| 1300030906 | 7天无理由退货（定制类不支持）|
| 7天无理由退货、七天无理由退货 | 1300030901 | 7天无理由退货 |

**注意**：
- 支持中文全角括号（）和半角括号()
- 必须完整匹配标签名称，不支持部分匹配
- 如果输入值不在上述列表中，默认使用 1300030895（不支持7天无理由退货）

## 常见问题

### Q1: 导入时提示字段不存在？
A: 确保已执行数据库迁移脚本，添加了 `no_reason_return_tag_id` 字段。

### Q2: 导出的标签ID还是错误？
A: 检查数据库中 `no_reason_return_tag_id` 字段的值，确保是有效的标签ID。

### Q3: 旧数据怎么办？
A: 数据库迁移脚本会自动将旧数据转换为标签ID格式。

### Q4: 如何验证转换是否正确？
A: 执行以下SQL查询：
```sql
SELECT 
    product_name,
    no_reason_return,
    no_reason_return_tag_id
FROM t_product
WHERE no_reason_return_tag_id IS NULL OR no_reason_return_tag_id = '';
```
如果有结果，说明有数据未正确转换。
