# 商品图片批量导出功能 - 按美团规范

## 功能需求

### 导出规范
1. **图片格式**：png、jpg、jpeg
2. **图片尺寸**：1200×1200px (1:1) 或 1200×800px (4:3)
3. **图片大小**：单张小于5M（超过自动压缩）
4. **批量限制**：
   - 文件夹最大1G
   - 一次最多300张图片

### 图片命名规则
支持三种命名方式：
- **商品名称**：`商品名称-1.jpg`、`商品名称-2.jpg`...
- **条形码**：`条形码-1.jpg`、`条形码-2.jpg`...
- **店内码/货号**：`店内码-1.jpg`、`店内码-2.jpg`...

**特殊字符处理**（Windows兼容）：
- `/` → `_`
- `*` → `^`

### 导出格式
- 所有图片放在同一个文件夹
- 打包成ZIP压缩包下载

## 实现步骤

### 1. 后端实现

#### 1.1 创建图片导出服务
文件：`meituan-backend/src/main/java/com/meituan/product/service/ProductImageExportService.java`

功能：
- 从OSS下载商品图片
- 按规范重命名
- 压缩超过5M的图片
- 打包成ZIP

#### 1.2 添加导出接口
文件：`meituan-backend/src/main/java/com/meituan/product/controller/ProductImageController.java`

接口：`POST /api/products/export-images`

参数：
```json
{
  "merchantId": 1,
  "namingType": "PRODUCT_NAME",  // PRODUCT_NAME | BARCODE | STORE_CODE
  "productIds": [1, 2, 3]  // 可选，不传则导出全部
}
```

#### 1.3 添加图片压缩依赖
在 `pom.xml` 中添加：
```xml
<!-- 图片处理库 -->
<dependency>
    <groupId>net.coobird</groupId>
    <artifactId>thumbnailator</artifactId>
    <version>0.4.19</version>
</dependency>
```

## 使用流程

1. 用户在商品管理页面点击"导出商品图片"
2. 选择命名方式（商品名称/条形码/店内码）
3. 可选择导出全部或选中的商品
4. 系统自动：
   - 从OSS下载图片
   - 按规范重命名
   - 压缩超过5M的图片
   - 打包成ZIP
5. 自动下载ZIP文件

## 技术细节

### 图片压缩算法
- 检测图片大小
- 如果超过5M，按比例压缩
- 保持宽高比
- 质量设置为0.85

### 文件命名处理
```java
// 示例：商品名称 "Apple iPhone 15 Pro/Max*"
// 处理后：Apple_iPhone_15_Pro_Max^-1.jpg
```

### 错误处理
- 图片下载失败：跳过该图片，记录日志
- 压缩失败：使用原图
- 超过300张：分批导出提示
